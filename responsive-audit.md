# Responsive audit — airfa-website vs original (sites.google.com/view/airfa-desde1895)

Goal: make the site pixel-match the original on **mobile** (390px) and **desktop** (1366px), all features working.
Tooling in `_cmp/` (gitignored): `batch.mjs` captures full-page screenshots of both sites; `shoot.mjs` single page.
Terminate when a cycle finds **no differences in both modes**.

Architecture note: homepage (`index.html`) is flow-based and fully reflowable. The 13 content pages are
absolute-positioned pixel canvases generated by `build.mjs` from `_scrape/layout/*.json`. The original
reflows them to single-column on mobile (heights ~2× desktop), so those pages need a reflow path.

---

## Cycle 1 — MOBILE diff (390px)

Baseline: site has **zero media queries**; everything renders at desktop scale.

### Global (all pages)
- **G1** Header shows full desktop nav crammed/overflowing; no hamburger. Original = `[☰] [crest] … [🔍]`.
- **G2** Page content overflows horizontally (homepage measured 857px wide vs original 390px).
- **G3** `brand-title` text "AIRFA - DESDE 1895" shown; original hides it on mobile (crest only).

### Homepage (index)
- **H1** Hero title `ACADEMIA ALMADENSE` at 96px overflows viewport / gets clipped; original wraps & scales down.
- **H2** Hero social row missing/misplaced on mobile.
- **H3** Carousel poster 520px fixed track overflows; original shows single contained poster.
- **H4** Section headings (131 ANOS 64px, PRÓXIMAS 50px, ATIVIDADES 40px) too large for 390px.

### Content pages (a-academia, historia, banda, … 13 pages)
- **C1** `.page-canvas` is `width:1366px` fixed → massive horizontal overflow on mobile.
- **C2** Original reflows these to single-column (e.g. historia 8772px tall mobile vs 5155 desktop).
      Needs a reflow rendering path; absolute coords can't match as-is.

### Fixes applied (Cycle 1)
- Added hamburger toggle markup + JS to header (index.html + build.mjs); moved search button out of nav.
- `@media (max-width:768px)` block: header→hamburger drawer, hide brand-title, responsive hero/headings/carousel/footer.
- Content pages: build.mjs now tags decorative rects/lines/boxes `.deco` and **sorts canvas elements into
  reading order (top,left)**; mobile CSS flips `.abs` absolute→static so the canvas reflows to one column,
  hides `.deco`, darkens on-band text, scales images/iframes/timeline.

### Cycle 1 RESULT (re-captured both modes)
- **DESKTOP: perfect — 0% height delta on all 14 pages.** build.mjs reorder did not regress the pixel canvas.
- **MOBILE: reflows correctly; most pages within ~10-17% of original height.** Outliers to fix next:
  - `index` -40% (1578 vs 2637) — homepage too short vs original.
  - `documentos` -49% (880 vs 1735) — content missing/under-rendered.
  - `banda` -28% (4369 vs 6109) — timeline accordion collapsed; original taller.
  - `estatutos` +17% — slightly tall.

## Cycle 2 — MOBILE refinement (desktop already matches)

Method: re-captured all pages; ran 5 parallel QA agents diffing each page's mobile clone vs original.

### Fixes applied
- **Buttons:** text with a colored box (`e.box`) is now passed its band color as `--band` and styled as a
  real centered button on mobile (`.on-band`). Fixed documentos "Relatório e Contas" / privacy buttons,
  which had been stripped to plain text.
- **Canvas layout:** switched mobile canvas from flex-column to block so images/buttons center and footer
  social icons flow horizontally instead of stacking.

### Verified results
- **DESKTOP: still exact pixel match** (documentos/orgaos/banda/actividades/hino heights identical to original)
  — none of the reflow/build changes regressed the desktop canvas.
- **MOBILE: close match on the text-heavy majority** (index, a-academia, historia, estatutos, biblioteca,
  banda, salas-de-espetaculo, cine-teatro, sala-de-cinema, actividades, documentos).

### Residual mobile gaps (not fully resolved)
- **R1 [card grids] orgaos-sociais, hino:** photo/name/role of each member reflow as *grouped by type*
  (all photos, then all names, then all roles) instead of per-card. Root cause: cards sit in a 2-D grid
  (4 cols × N rows); top-sort can't keep a card together, column-major would break the per-section grouping,
  and DOM-regrouping would break the pixel-perfect desktop. Proper fix = **dual render** (emit a separate
  mobile single-column flow per page). All content is present, just ordered photos→names→roles.
- **R2 [content drift] contactos:** the *live original* now shows GPS coords + "Secretaria" opening hours;
  our scrape (`_scrape/`) predates that and shows the street-address block. This is a content change in the
  source, not a responsive bug.
- **R3 [minor] actividades:** the "ACTIVIDADES CULTURAIS" band heading between the two tables is faint/missing.
- **R4 [minor] cookie box:** our fixed cookie replica overlaps the last items on very short pages in
  screenshots (both sites show a cookie box; ours is `position:fixed`).

### Status
- Desktop: **matches (pixel-exact).**
- Mobile: **responsive and close**, with residuals R1–R4. Literal pixel-parity on the reflowed card grids
  (R1) needs the dual-render follow-up; R2 is a source-content change.

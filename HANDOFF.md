# AIRFA website replica ‚Äî handoff

A pixel-faithful static reproduction of the AIRFA Google Site
(`https://sites.google.com/view/airfa-desde-1895-copia`). 14 pages: the homepage
+ 13 sub-pages. Served as plain static files.

## How to run
```bash
cd <repo root>
python3 -m http.server 8765 --bind 127.0.0.1   # then open http://127.0.0.1:8765/
```
Open `index.html` directly in a browser too ‚Äî everything is static.

## Architecture
- **`index.html`** ‚Äî homepage, hand-built and pixel-matched (hero, auto-rotating
  carousel, sections, footer). NOT generated by build.mjs.
- **13 sub-pages** (`a-academia.html`, `historia.html`, ‚Ä¶) ‚Äî **generated** by
  `build.mjs` from captured geometry. Run `node build.mjs` to regenerate them.
- **`styles.css`** ‚Äî shared styling for all pages.
- **`assets/`** ‚Äî all images:
  - `assets/banner-bg/` ‚Äî page-title banner background images
  - `assets/geo/` ‚Äî per-page content images (positioned by geometry)
  - `assets/galeria/gallery.png` ‚Äî the expanded "Galeria de Presidentes" gallery
  - `assets/timeline/` ‚Äî `sm_YYYY.png` (collapsed band) + `big_YYYY.png` (expanded) per year
  - `assets/crest.png`, `hero-theater.jpg`, posters, social icons

### How the sub-pages are built (important)
Each sub-page is reconstructed by **absolute positioning from the original's exact
geometry**. The pipeline:
1. `_scrape/layout.mjs` visits each original page, captures every element's
   position/size/computed-style/links + downloads images ‚Üí `_scrape/layout/<slug>.json`.
2. `build.mjs` reads those JSONs and emits absolutely-positioned HTML. Each element
   lands at its exact (x, y) with the original's fonts, so text wraps identically.
   Anchor = 341px (56px header + 285px banner); `documentos` uses 896 (tall banner).
3. Banners: rendered as background-image + **live HTML title** (fixed 80px / 64px on
   cine-teatro) + coral underline + ~45% dark overlay ‚Äî so the title never scales on
   wide windows (this was a real bug; baked-in titles ballooned on >1366px screens).

### Interactions (all working)
- **Nav dropdowns** (A Academia, Salas de Espet√°culo) ‚Äî CSS hover.
- **Maestro collapsible** (banda) ‚Äî click expands the Francisco Pinto bio; content
  below reflows. Config in `COLLAPSIBLES.banda` in build.mjs.
- **Galeria de Presidentes** (historia) ‚Äî click reveals the president-timeline gallery
  image. `COLLAPSIBLES.historia`.
- **A Banda no Tempo** (banda) ‚Äî accordion of 11 year bands; click a year to swap the
  darkened strip for the enlarged photo and push content down. `TIMELINE.banda`.
- The collapsible/accordion reflow JS is `COLLAP_JS` in build.mjs (parses base tops,
  shifts elements below expanded widgets, supports multiple open at once).

### Regenerating from the original
The `_scrape/` folder (gitignored) holds the extraction tooling and reference
screenshots. Key scripts:
- `layout.mjs` ‚Äî main geometry extractor (`node _scrape/layout.mjs [slug] [vpHeight]`).
- `bannerinfo.mjs` ‚Äî banner bg + title styles. `measure_underlines.py` ‚Äî underlines.
- `capture_galeria.mjs`, `capture_tl_big.mjs` ‚Äî expanded-content captures.
- `shot.mjs` ‚Äî screenshot a URL full-page (`node _scrape/shot.mjs <url> <out.png> 1366`).
All use Puppeteer driving `/usr/bin/google-chrome-stable`.

### Regeneration caveat (READ before running `node build.mjs`)

## FIXES APPLIED ‚Äî 2026-06-21 (session 3)
1. **Homepage hero banner ‚Äî coral line + framing.** Three problems (all `styles.css`
   `.hero*`): (a) the coral accent rendered as a *strikethrough* through the title
   because `.hero-title::before` was at `top:50%` ‚Äî moved to `top:calc(100% + 6px)`
   so it is an UNDERLINE below "ACADEMIA ALMADENSE" (matches the original; the colour
   `#ff5252` and width `--content` were already correct). (b) Hero was a fixed
   `840px`; the original banner is a full-viewport `100vh` image *behind* the header.
   Changed to `height:100vh;margin-top:-56px` so the photo spans the whole viewport
   and its top 56px sit under the sticky white header ‚Äî pixel-matches the original and
   is responsive. (c) Focal `center 42%` ‚Üí `center 50%`. The hero image
   (`assets/hero-theater.jpg`) was already byte-identical to the original's banner.
2. **Sub-page banner backgrounds ‚Äî RESOLVED issue 2 (the 56px-header-overlap crop).**
   Root cause was NOT a wrong source image (every downloaded original banner is now
   byte-identical to ours) ‚Äî it was the crop geometry. The original small banner is a
   **340px-tall** `cover`/`50% 50%` image whose **top 56px are hidden behind the
   header**; the replica was fitting `cover` to the 285px box *below* the header, so
   the crop landed ~28px too high. Fix: on each affected `.banner-bg`, replace
   `inset:0` with `left:0;right:0;bottom:0;height:341px` ‚Äî the bg is now `cover`-fit to
   341px and bottom-anchored, so `overflow:hidden` on `.page-banner` clips the top 56px
   exactly like the header does. Applied to: a-academia, historia, estatutos,
   biblioteca, contactos, orgaos-sociais, hino, sala-de-cinema, and documentos
   (documentos also had its title/rule nudged +50px to `top:414`/`518` to match, and
   uses `height:896px` = its 840px banner + 56). Verified each against the live site
   (mean banner-region diff dropped from ~27‚Üí~10 for the parchment pages). The
   `=w2000` lh3 URL trick downloads the real bg: read the element's computed
   `background-image` `googleusercontent` URL from the live DOM, then `curl` it (tokens
   expire in ~minutes, so probe-and-download in one pass).
   - **Left as-is (already optimal):** cine-teatro (already 2% diff), banda (its image
     has no vertical crop slack ‚Äî position-y is a no-op; the ~big residual is JPEG
     recompression of a busy group photo), salas-de-espet√°culo (a position sweep
     confirmed the existing `50% 100%` is the true minimum). Don't "fix" these ‚Äî the
     anchor change made salas measurably worse.

# AIRFA website replica — handoff

A pixel-faithful static reproduction of the AIRFA Google Site
(`https://sites.google.com/view/airfa-desde-1895-copia`). 14 pages: the homepage
+ 13 sub-pages. Served as plain static files.

## How to run
```bash
cd /home/loureiro/dev/airfa-website
python3 -m http.server 8765 --bind 127.0.0.1   # then open http://127.0.0.1:8765/
```
Open `index.html` directly in a browser too — everything is static.

## Architecture
- **`index.html`** — homepage, hand-built and pixel-matched (hero, auto-rotating
  carousel, sections, footer). NOT generated by build.mjs.
- **13 sub-pages** (`a-academia.html`, `historia.html`, …) — **generated** by
  `build.mjs` from captured geometry. Run `node build.mjs` to regenerate them.
- **`styles.css`** — shared styling for all pages.
- **`assets/`** — all images:
  - `assets/banner-bg/` — page-title banner background images
  - `assets/geo/` — per-page content images (positioned by geometry)
  - `assets/galeria/gallery.png` — the expanded "Galeria de Presidentes" gallery
  - `assets/timeline/` — `sm_YYYY.png` (collapsed band) + `big_YYYY.png` (expanded) per year
  - `assets/crest.png`, `hero-theater.jpg`, posters, social icons

### How the sub-pages are built (important)
Each sub-page is reconstructed by **absolute positioning from the original's exact
geometry**. The pipeline:
1. `_scrape/layout.mjs` visits each original page, captures every element's
   position/size/computed-style/links + downloads images → `_scrape/layout/<slug>.json`.
2. `build.mjs` reads those JSONs and emits absolutely-positioned HTML. Each element
   lands at its exact (x, y) with the original's fonts, so text wraps identically.
   Anchor = 341px (56px header + 285px banner); `documentos` uses 896 (tall banner).
3. Banners: rendered as background-image + **live HTML title** (fixed 80px / 64px on
   cine-teatro) + coral underline + ~45% dark overlay — so the title never scales on
   wide windows (this was a real bug; baked-in titles ballooned on >1366px screens).

### Interactions (all working)
- **Nav dropdowns** (A Academia, Salas de Espetáculo) — CSS hover.
- **Maestro collapsible** (banda) — click expands the Francisco Pinto bio; content
  below reflows. Config in `COLLAPSIBLES.banda` in build.mjs.
- **Galeria de Presidentes** (historia) — click reveals the president-timeline gallery
  image. `COLLAPSIBLES.historia`.
- **A Banda no Tempo** (banda) — accordion of 11 year bands; click a year to swap the
  darkened strip for the enlarged photo and push content down. `TIMELINE.banda`.
- The collapsible/accordion reflow JS is `COLLAP_JS` in build.mjs (parses base tops,
  shifts elements below expanded widgets, supports multiple open at once).

### Regenerating from the original
The `_scrape/` folder (gitignored) holds the extraction tooling and reference
screenshots. Key scripts:
- `layout.mjs` — main geometry extractor (`node _scrape/layout.mjs [slug] [vpHeight]`).
- `bannerinfo.mjs` — banner bg + title styles. `measure_underlines.py` — underlines.
- `capture_galeria.mjs`, `capture_tl_big.mjs` — expanded-content captures.
- `shot.mjs` — screenshot a URL full-page (`node _scrape/shot.mjs <url> <out.png> 1366`).
All use Puppeteer driving `/usr/bin/google-chrome-stable`.

## Verification status
All 14 pages match the original **height exactly** at 1366px width. Per-page pixel
diff (fuzz 12%, mostly font-antialiasing + photo recompression, NOT layout):
a-academia 2.8%, historia 4.2%, estatutos 3.5%, orgaos 4.0%, hino 2.9%,
biblioteca 7.9%, documentos 16.8%, actividades 15.0%, banda 12.8%, salas 7.4%,
cine-teatro 4.9%, sala-de-cinema 6.6%, contactos 6.5%. (Higher numbers are the
image-heavy pages — banner photo + tables + timeline photos recompress, so many
pixels differ slightly while looking identical.)

## KNOWN ISSUES / REMAINING WORK
1. **Hino verses not filled in.** The page is fully built (composer/lyricist photos,
   names, intro, score, both videos) with a ready-to-fill container in `hino.html`
   (`id="hino-letra"`, marked with a `cole aqui as estrofes` comment). The verses
   themselves must be pasted in there by the site owner (one `<p>` per stanza).
   I did not transcribe them.
2. **Textured banner backgrounds — framing slightly off.** On the parchment-family
   pages (a-academia, historia, estatutos, biblioteca) and contactos/documentos, the
   banner BACKGROUND image crop/zoom differs a little from the original. The original
   serves the banner bg via stacked/lazy image versions whose tokens change on every
   load, so the exact pre-framed source image couldn't be reliably downloaded. The
   title + overlay are correct; only the bg framing differs. FIX: download those
   banner bg images from the Sites editor and drop into `assets/banner-bg/<slug>.<ext>`,
   then rebuild. Photo banners that loaded cleanly (banda, cine-teatro, salas) match.
3. **Pixel-perfection is tuned to 1366px width.** Sub-pages use absolute positioning
   anchored to 1366. They are NOT responsive/reflowing for arbitrary widths (the
   original Google Site is responsive). Titles are fixed-size so they don't balloon,
   but full responsive reflow is not implemented.
4. **Higher-diff pages** (documentos/actividades/banda) are dominated by image
   recompression, but worth a final eyeball against the live site.
5. **`_scrape/` is gitignored** — the layout JSONs needed by `build.mjs` live there.
   If someone clones fresh, the generated `*.html` are committed and work standalone,
   but re-running `build.mjs` needs the `_scrape/layout/*.json` files (regenerate via
   `node _scrape/layout.mjs`). Consider committing `_scrape/layout/` if reproducibility
   from source is desired.
